Езикът Java

Java е съвременен обектно-ориентиран език за програмиране с общо предназначение. На Java се разработва изключително разнообразен софтуер: офис приложения, уеб приложения, настолни приложения, при­ложения за мобилни телефони, игри и много други.

Java е език от високо ниво, който прилича на C# и C++ и донякъде на езици като Delphi, VB.NET и C. Java програмите са обектно-ориентирани. Те представляват съвкупност от дефиниции на класове, които съдържат в себе си методи, а в методите е разположена програмната логика. Повече детайли за това какво е клас, какво е метод и какво представляват Java програмите ще научите в следващата глава.

В днешно време Java е един от най-популярните езици за програмиране. На него пишат милиони разработчици по цял свят. Най-големите световни софтуерни корпорации като IBM, Oracle и SAP базират своите решения на Java платформата и използват Java като основен език за разработка на своите продукти. Сред "големите" Java не се ползва единствено от Microsoft, тъй като те разра­ботват и поддържат собствена платформа, подобна на Java платформата, наречена .NET Framework и език подобен на Java, наречен C#.

Езикът Java е първоначално разработен и поддържан от Sun Microsystems, но през 2006 г. Java плат­формата става проект с отворен код и в момента се поддържа и развива от световната Java общност съвместно със Sun. Благодарение на отворения код популярността на Java постоянно се увеличава и броят Java разработчици непрекъснато расте.

Езикът Java се разпро­странява заедно със специална среда, върху която се изпълнява, наре­чена Java Runtime Environment (JRE). Тази среда включва т. нар. Java виртуална машина (JVM) и пакет стандартни библиотеки, предоставящи базова функционалност. Благодарение на нея Java програмите са преносими и след като веднъж бъдат написани, след това почти без промени могат да работят на стотици хардуерни платформи и операционни системи.

Java платформата

Езикът Java не се разпространява самостоятелно, а е част от платформата Java. Java платформата най-общо представлява среда за разработка и изпълнение на програми, написани на езика Java. Тя се състои от езика Java, виртуалната машина на Java (JVM), която изпълнява Java програ­мите и от съвкупност от стандартни библиотеки и инструменти за разра­ботка, като например компилаторът, който превръща Java програ­мите в разбираем за виртуалната машина междинен код (Java bytecode).

За изпълнението на програми, написани на Java, е необходимо да имате инсталиран т. нар. Java Runtime Enviroment (JRE). Това е специален софтуер, част от Java платформата, който съдържа виртуалната машина и стандартните Java библиотеки и се поддържа за различни хардуерни платформи и операционни системи. JRE е софтуер, който крайният потребител инсталира еднократно върху компютъра си, за да може да работи с Java. JRE не е стандартна част от Windows и трябва да се инсталира допълнително, точно както “Adobe Flash Player" се инсталира допъл­нително във вашия уеб браузър, за да отваряте уеб сайтове с Flash.

За разработката на Java приложения е необходимо да имате инсталиран Java Development Kit (JDK). Това е пакет инструменти, с които вие като програмисти можете да пишете Java програми, да ги компилирате и изпълнявате. JDK не е необходим на крайния потребител, а само на Java разработчиците. Крайните потребители ползват JRE.

Същевременно Java е изключително прост и лесен за научаване език (за разлика от C и C++). Нормално е да започнем от език, който е подходящ за начинаещи и се ползва много в практиката. Когато някой програ­мира некадърно, лекторите му показват къде и какво греши, за да тръгне в правилната посока. Качеството на усвояване на учебния мате­риал и въобще писането на качествен програмен код и използването на утвърде­ните практики в софтуерното инженерство са основен приоритет при всички обучения в НАРС. Заедно с техническите умения курсистите се обучават и на добрите практики в програмирането и въобще в софтуер­ното инженерство.

Лекторите отделят индивидуално внимание на всеки един курсист, независимо дали той има въпроси, дали има трудности или се справя отлично. При всички случаи преподавателят разглежда всеки какво е написал по всяка от поставените задачи и дава персонални наставления за подобряване на кода, идеи за решаване на задачите и препоръки за преработка, когато се установи некачествен код.

Изпити

Изпитите в НАРС биват теоретични и практически, като основната тежест пада върху практическото решаване на изпитни задачи.

Теоретичните изпити представляват въпроси върху изучавания учебен материал, на които студентите отговарят чрез посочване на верния отговор сред списък от възможни отговори.

Практическите изпити са по-важни и по-сериозни. При тях на всеки курсист се възлагат няколко задачи, за които той трябва да напише решение. Курсистът разполага с компютър, на който пише решенията във вид на програма или малък софтуерен проект. Докато курсистите пишат, могат да използват всякакви учебни материали и Интернет (ситуация много подобна на реалната работа в софтуерна фирма), но нямат право да получават чужда помощ от никого (по телефон, чат, e-mail и всякакви други начини). При изтичане на определеното време за работа преподава­телите проверяват задачите на всички и поставят оценки. При незадово­лително представяне съответният студент се явява на поправка след 1 или 2 седмици. Курсистите, които системно показват слаби резултати, се отстраняват от обученията.

Практически проекти

Практическите проекти в Академията представляват цялостен проект за разработване на софтуерна система, близка до реалните системи, по които се работи в индустрията, само че с по-малък обхват. Практическите проекти включват не само програмиране, но и всички останали елементи от софтуерното инженерство: анализ на изискванията, изготвяне на спецификация, изготвяне на прототип на потребителския интерфейс, моделиране на база данни, построяване на слой за достъп до данните, изграждане на уеб базиран потребителски интерфейс, изграждане на desktop базиран потребителски интерфейс, тестване, отстраняване на бъгове, работа с хранилище за сорс код с контрол на версиите (source control repository) и съвместна работа в екип, заедно с колеги от курса.

Практическите проекти дават на курсистите реален опит при разработката на софтуер, тъй като само уменията по програмиране не са достатъчни за работа в софтуерна фирма. Практическите проекти подготвят курсистите за реалния живот на софтуерния инженер, за неговата ежедневна работа.

Практически стажове

Стажовете в НАРС или партньорски фирми са предназначени само за отбрани курсисти, които се справят по-добре от останалите и правят силно положително впечатление на преподавателския екип. Понякога на стаж биват приемани повече от половината курсисти, а понякога само 2-3 от група. Възмож­ностите за стажове зависят от заявките на фирмите за стажанти и имат променлив характер.

По време на стажовете курсистите се включват в разработката на реален софтуерен проект, като идват на работа в съответната фирма, където са на стаж, и участват в екипа на разработка наравно с останалите служи­тели от фирмата. По този начин курсистите натрупват ценен реален опит за работа по реални проекти в истинска софтуерна компания. Ако ком­панията оцени високо работата на стажанта, обикновено му предоставя и заплащане за периода на стажа (по нейна преценка).

Профили на курсистите

Профилите на курсистите са вътрешни документи, които НАРС поддържа за всеки курсист, който се обучава в Академията. В тях се съдържат автобиографиите на курсистите, информацията от проведеното в началото интервю, данните от психологическия профил на курсиста, резултати от всички изпити, на които се е явявал, включи­телно и сорс кода на прог­рамите, които е написал по време на изпитите. Профилът съдържа още мнение за курсиста от преподавателския състав и история на интервю­тата, на които е ходил.

До профилите на курсистите достъп имат фирми-работодатели, които са партньори на Академията. Към момента на писането на този текст (октомври 2008) всички фирми, с които НАРС работи надвишават 100. Сред тях са световноизвестни софтуерни компании като SAP, VMWare, Telerik, Software A. G., Seeburger Informatik, Sirma Group и много, много други ИТ фирми. Всички тези фирми предлагат позиции за работа на курсистите от НАРС, които си харесат.

Интервюта за работа

Интервютата за работа, организирани от НАРС са част от цялостния процес, в който се включва всеки един курсист, започвайки безплатното си обучение в Академията. По време на обучението в НАРС и след неговото завършване курсистите получават покани за интервюта с рабо­тодатели, на които са длъжни да се явят и да се представят максимално добре. Всеки курсист получава инструктаж как да се подготви и как да се държи на интервюто, какви въпроси да очаква, какво е добре и какво не е добре да приказва на евентуалния си работодател и други полезни препоръки. Повечето интервюта протичат в офисите на работодателите.

По време на едно интервю представители на работодателите задават технически и други въпроси на кандидатите, опитват се да преценят до къде се простират техните знания и умения, както и да извлекат инфор­мация за личностните им качества. Решенията дали да отправят предло­жение за работа към всеки един кандидат и при какви условия се вземат от самата фирма-работодател след интервюто. Условията могат да са много различни и зависят както от самия работодател и спецификата на неговите проекти, така и от представянето на курсиста по време на персоналното интервю.

Работодателите не канят на интервю всички курсисти, а само някои от тях в зависимост от много фактори: автобиография, изпитни резултати, специфични личностни качества, владеене на чужди езици, евентуален предишен опит и др. Фирмите-работодатели имат много различни (и понякога дори странни) кри­терии кого да повикат на интервю и на кого да предложат работа. Различните фирми предпочитат различни профили курсисти. Не винаги най-силните от курса първи започват работа. Обик­новено всички започват работа до завършване на курса или най-много месец след това.

Когато курсист започне работа, той продължава да посещава курса докато не завърши успешно. Работодателите следят стриктно резултатите на техните курсисти и могат да променят отношенията с курсиста (включи­телно и да коригират заплатата му) според постигнатите резултати на изпитите в Академията.

В редки случаи може да се случи някой курсист да не бъде предпочитан и канен на интервюта, както и да не бъде харесван от работодателите по време на интервютата. Причините за това обикновено са в самия курсист, който с поведението си показва безотговорност и излъчва усещане, че не му се работи. Например, ако на интервю за работа кажете нещо в стил "аз искам малко да поработя във вашата фирма, че да понатрупам опит и след това ще заминавам за чужбина", няма как да ви предложат оферта. Друг пример е изказване в стил "ами чувал съм, че програмистите взимат по няколко хиляди лева заплата, аз искам и аз толкова като за начало". Такива неадекватни очаквания и изказвания гарантират неуспех по интервютата. В НАРС се стараем да обучаваме курсистите как да се държат на интервютата, за да получат работата, за която кандидатстват, но успехът зависи най-вече от самия курсист, а не от обученията.

Ако кандидатът за работа се държи неадекватно по интервю­тата, е нормално фирмите да не му предлагат работа. В НАРС се стремим въобще да не приемаме в курсовете несериозни и неамбицирани курсисти, хора с неадекватни очаквания и хора, на които не им се работи, но винаги се случва някой "тарикат" да се промъкне. Такива случаи са рядкост и се решават инди­видуално.

Договорите с НАРС

Вече споменахме, че Национална академия по разработка на софтуер не е благотворителна организация. Обучението в Академията се регламентира от договори, които обучаемите сключват с НАРС, с които те се ангажират да се отнасят сериозно към учебния процес и да започнат работа след завършване на обучението си или още от преди това.

Фирмите-работодатели, които спонсорират безплатните обучения, също изискват подписване на договор, с който служителят се ангажира да работи при своя работодател за най-малко 1-2 години, с което оправдава неговата първоначалната инвестиция. Условията по договорите са индивидуални и различни за различните работодатели, но като цяло работодателите се стремят да предлагат адекватни за пазара условия на работа и добро ниво на заплащане, за да задържат кадрите, в които инвестират дългосрочно. Фирмите, с които НАРС работи, имат висок имидж, изграден върху коректни отношения със своите служители, клиенти и партньори.

Обученията в НАРС са наистина безплатни, но само, ако наистина ви се учи и ви се работи сериозно. Ако нямате желание и сили да отделите цялото си време за курсовете и след това да работите сериозно на пълен работен ден, помислете за платения вариант на курсовете.

Има ли платено обучение?

Всичко, което до момента ви разказахме, се отнася за безплатните обучения, при които курсистите учат, без да плащат, но се ангажират след завършване на курса (или още по време на обучението си) да започ­нат работа в софтуерна компания (която реално поема таксите по тяхното обучение) и да работят в нея поне 1-2 години (зависи от конкретната оферта).

В НАРС се предлага и платен прием, при който отпадат ограниченията и ангажи­ментите към Академията и към работодателите. Всеки може сам да прецени дали да се обвърже с фирма-работодател, която да спонсорира обучението му, или да заплати самостоятелно таксите за курсовете, по които се обучава, и след това да не работи или да работи, където си поиска (включително и в чужбина). Това е въпрос на личен избор и, разбира се, на възможностите, с които всеки разполага.

Курсовете за начинаещи

Курсовете за начинаещи в Академията обхващат в най-общи линии учебния материал от настоящата книга. Те са с продължителност 1 месец, по 2-3 пъти в седмицата по 4 часа на ден. Те не ви дават професия, нито ви осигуряват работа, но ви дават основата, от която да тръгнете в изучаването на професията "софтуерен инженер".

Въведителните курсове се предлагат в няколко варианта, които са долу-горе равностойни като обхват на учебния материал и обем на натрупаните знания, но се различават по езика за програмиране. Това са курсовете за начинаещи към настоящия момент (октомври 2008):

-     "Въведение в програмирането със C#" – този курс обхваща учебния материал от настоящата книга, но с езика C#, и ви подготвя за следващото ниво на обучение. След него може да се обучавате за софтуерен инженер в направления Microsoft .NET, Java или други платформи.

Курсовете за начинаещи служат като филтър, през който преминават само курсистите, на които материята им се отдава, и се отнасят сериозно към обучението. Обикновено не повече от половината, които са приети за обучение по въведителните курсове в Академията, успяват да завършват успешно. Само те продължават по-нататък, в курсовете за напреднали, и достигат в някой момент до достатъчно знания и умения, за да получат предло­жения за работа.

Стъпка 6: Интервюта за работа

Още по време на курса за всеки курсист се изготвя и поддържа профил, съдържащ всички негови резултати от изпити, автобиография и други документи. По този профил софтуерните компании преценяват кои курсисти да поканят на интервю.

Един курсист може да бъде поканен на едно или повече интервюта, но силните студенти от курса обикновено се справят още на първото интервю и постъпват на работа.

При покана за интервю трябва добре да се подготвите. Това е вашето представяне, от което зависи дали ще ви предложат работа. Трябва добре да проучите фирмата, в която ви канят: с какво се занимава, по какви проекти работи, какви технологии използва и т. н. Трябва да се държите адекватно на самото интервю, за което ще получите препоръки от НАРС. Освен техническите задачки, ще ви питат каква заплата очаквате и други въпроси. Много е важно да не се "изцепвате" с неадекватно очакване за заплащане. Никой не започва с директорска позиция и огромна заплата, така че имайте търпение. Ако работите сериозно и давате сериозен принос за проектите, в които ви включват, заплатата ви постепенно ще расте. Така е във всяка фирма.

Как протича едно интервю за работа?

Едно типично интервю за работа протича в следните стъпки:

1.  Интервюиращия ви се представя и ви разказва накратко за фирмата и отдела, за който кандидатствате. Обяснява ви за проек­тите и технологиите, които използват и други общи приказки.

2.  Вие се представяте накратко с няколко думи. Може да споменете какво сте учили, от кога се занимавате с програмиране, какво ви е запалило и т.н. Трябва да покажете силен интерес към професията.

3.  Интервюиращият (често пъти може да са няколко души) ви задава техни­чески въпроси. Целта е да провери знанията ви. Примерно може да ви пита "какво е хеш-таблица и как работи" или "как работи полиморфизмът в ООП", може да ви накара да напишете парче код, примерно да имплементирате свързан списък или да разбъркате тесте карти в случаен ред. Това е първата задачка – тя обикновено не е свързана с технологии, а просто проверява дали имате алгорит­мично мислене.

Каквото и да ви питат, трябва да се мъчите да стигнете до някакво решение. Вземете лист и химикал, направете скица на задачата и разсъждавайте на глас. Не е толкова важно дали ще решите задачата, а дали ще подходите към нея правилно. В главата "Как да решаваме задачи по програми­ране?" има много полезни съвети как да подхождате при решава­нето на задачи по програ­миране. След­вайте тези съвети и на интервюто за работа.

След това може да ви питат за технологични задачки, примерно да моделирате проста база данни, да напишете някаква SQL заявка, могат да ви питат за XML, за HTML, за уеб технологии и за много други неща, които сте учили или ще учите в курса на НАРС. Трябва винаги да търсите решение, да разсъждавате, дори да не знаете отговора. Ако почнете да мислите дълго, без да говорите, това е лош знак. Трябва да мислите и разсъждавате на глас!

4.  Следват нетехническите въпроси. Те могат да са изключително разнообразни, дори понякога странни. Примерно може да ви помолят да се опишете с 3 думи, или да кажете 3 добри и 3 лоши ваши качества. Очаквайте такива въпроси и се подгответе. Могат да ви питат и странни въпроси, като примерно дали спортувате, дали играете футбол и на каква позиция. Целта на всички тези въпроси е най-вече да ви преценят дали ще работите добре в екип.

Сред нетехническите въпроси ще ви питат къде сте учили, какво сте научили, къде смятате да учите и как си представяте разви­тието си след няколко години. Подгответе се за такива въпроси. Винаги ги задават. Най-голямата глупост, която можете да кажете е нещо в стил "искам да поработя малко, че ще заминавам да уча в чужбина и ми трябват пари" или "не ми се работи, ама нямам пари" или "все ми е тая какво ще работя, само да плащат". Отговор, заради който можете да си тръгнете от интервюто е нещо в стил "мисля след 1 година да стана шеф на отдел, че не може само да програмирам". Дори да не сте скромен, проявете скромност на интервюто! Ще ви помогне.

Много е важно да вдъхвате усещане в отсрещната страна, че имате огромно желание да работите и то точно това, което ви предлагат (дори да не е вярно!) и че сте готови да четете много, да учите и да се справите, защото това е мечтаната за вас позиция и това е най-добрата фирма за вас. Трябва да казвате не само, че много ви се работи, а че много ви се работи точно в тази фирма и точно на тази позиция и че точно тези проекти, за които ви разказват, са ви супер интересни. За целта трябва предварително да проучите фирмата, в която отивате на интервю – да разгледате техния уеб сайт, да разпитате познати, да се поровите из Интернет и т.н.

5.  В един момент ще ви питат колко пари очаквате да получавате. Винаги задават този въпрос. Може да отговорите "вие преценете колко да ми предложите". Ако след това не ви отправят предло­жение, кажете някакво разумно число. Пригответе се за този въпрос. Абсо­лютно е сигурно, че ще ви го зададат!

Този въпрос винаги е труден. От една страна не искате да се прецакате и да поискате по-малко, отколкото те биха ви дали, а от друга – не искате да ви откажат работата само защото сте се полакомили за пари. Наистина, по-важното за вас е да започнете работа, отколкото да започнете на висока заплата. Ако не ви оценяват, след като понатрупате опит лесно ще си намерите по-добра работа, нали? Сега най-важното е да не се изцепвате – да поискате малко пари и да обясните, че очаквате повишаване с времето според справянето ви с поставените задачи. Това е най-нормалното нещо, което се случва в една фирма – всички ново­бранци започват от ниска заплата и с времето им я вдигат.

6.  В един момент ще ви питат дали имате някакви въпроси. Има два начина да сгрешите тук: да не питате нищо или да започнете да предявявате големи претенции. И в двата случая ще направи лошо впечатление и ще си намалите шансовете да ви предложат работа.

Ако не питате нищо, това означава, че не сте комуникативен (което ще ви пречи да работите в екип) или че не ви интересува нищо, стига да имате работа и да получавате заплата (което означава липса на мотива­ция).

Ако почнете да отправяте претенции за какво ли не, това е много лошо! Не е добре изкарвате на преден план въпроси като "колко е отпуската при вас", "има ли кафе в офиса", "има ли тенис маса" и подобни. Такива въпроси показват, че не се интересувате от работата, а само от забавления, пари и облаги. Никой не иска такива служители.

Бъдете внимателни с претенции в стил "аз съм студент и в сесията няма да идвам на работа". Това наистина е въпрос, който трябва да поставите, но го поставете внимателно, примерно кажете така: "Тъй като съм студент, около сесията става напрегнато с изпитите. Имате ли възможност в такива случаи, ако проектът позволява, да отсъствам по малко от работа. Аз, разбира се ще предупредя за това месец по-рано, но ме интересува как решавате проблема във вашата фирма, тъй като сигурно има и други студенти".

Не задълбавайте много по темата "пари, бонуси, осигуровки". Това показва, че за вас парите са най-важното и че ще напуснете фир­мата веднага, щом някой ви предложи малко по-висока заплата. Във всяка фирма се надяват да наемат служител дългосрочно. Трябва да правите впечатление, че ви е много интересно и че сте сериозни и ще работите дълго в компанията и ще се развивате вътре в нея дългосрочно.

Питайте въпроси около проекта, около технологиите и около работата. Това показва, че се интересувате какво ще работите, т. е. мотивирани сте и ви се работи. Питайте примерно "ако започна евентуално при вас, какъв ще ми е първият проект". Това е въпрос, който е 100% положителен и ви представя в добра светлина. Може да го запомните като "дежурния въпрос". Може да питате и други въпроси, свързани с работата и технологиите, примерно "при вас Oracle ли ползвате или друга база данни?". Може да питате каква е структурата на екипа, кой оставя задачите, по какъв начин, какво хранилище за сорс код се ползва, на какви езици и платформи се пише, колко голям е един екип и т.н. Това са все въпроси, които показват, че се интересувате какво ще работите, а не какви са облагите (пари, отпуски и подобни).

7.  Накрая интервюиращият ще ви благодари за срещата и ще ви обещае да се свърже с вас, веднага, след като обсъдят с колегите дали могат да ви отправят предложение за работа. Дори да ви харесат, няма да ви кажат "нает сте, честито". Ще трябва да изчакате. Има и други кандидати и те трябва да преценят.

Описаният сценарий малко или много се повтаря на всяко едно интервю за работа, така че пригответе се за него. Очаквайте въпросите и ситуа­циите, които описахме по-горе и това ще увеличи значително шансовете ви да преминете успешно интервюто.

Имайте адекватни очаквания за заплащане и се дръжте скромно. Когато ви питат технически въпроси, разсъжда­вайте на глас и си рисувайте проблема на лист хартия или на дъската. Питайте за проектите и за работата, а не за заплатата и за отпуските. Бъдете учтиви и възпитани. Не прекъсвайте другите докато говорят. Не правете изцепки!

Някои път силни кандидати се притесняват по интервютата и по тази причина се държат несвойствено, което им пречи да се справят добре. За да не се притеснявате на интервюто, можете да тренирате интервюта с ваши познати или просто да отворите обявите в Интернет и да подадете документи в няколко фирми. Така ще можете да отидете на няколко интервюта за работа и да натрупате опит. Може пък да ви одобрят. Тогава ще им обясните за Академията, курсовете и всичко останало. Запомнете какви въпроси ви питат и помислете какво трябва да отговаряте. Ако нe знаете кои са добрите отговори, питайте.

Стъпка 7: Постъпване на работа

Няколко дни след интервюто можете да очаквате предложение за работа. Ще ви се обадят и ще ви предложат договор за работа. Най-добре е да приемете, освен, ако няма нещо много фрапиращо в условията. Дори и условията да не са блестящи, важно е да започнете работа. Започването на първа работа е най-трудно. След като натрупате опит, можете да преговаряте за по-добри условия или да се преместите в друга фирма. Всички фирми търсят специалисти с опит.

Може да ви предложат пробен период. Това означава или че не са убедени, че ще се справите с работата, която ще ви възложат, или просто такива са процедурите им. И в двата случая е най-добре да се съгласите, дори да ви предложат по-ниско заплащане за първия месец или два, когато тече пробният период. При всички положения вие първият месец на една нова работа основно ще се запознавате с проекта и с технологии­те и реално няма да свършите работа, с която да си изкарате заплатата, така че помислете и за това. Фирмата ви плаща таксата за обучение, инвестира във вас в дълго­срочен план, с ясната идея, че в началото повече ще учите и ще навлизате в материята, отколкото ще допринасяте за проек­тите. След време ще навлезете в работата и ще станете наистина полезни на фирмата. Едва тогава тя ще има някаква възвращаемост. Затова не гледайте на пробния период само като една добра причина да ви плащат по-малко в първите месеци.

Най-вероятно от фирмата ще ви повикат на още една среща. Това е моментът да уточните детайлите по вашите отношения: ангажиментът към фирмата, осигуровките, отпуските, работното време и всякакви други въпроси. Можете да уточнявате по тези въпроси без притеснения, сега не сте на интервю и това няма да направи лошо впечатление.

Поглед към съдържанието на книгата

Нека сега разгледаме накратко какво ни предстои следващите глави на книгата. Ще разка­жем по няколко изречения за всяка от тях, за да знаете какво ви очаква да научите.

Глава 1. Въведение в програмирането

В главата "Въведение в програмирането" ще разгледаме основните термини от програмирането и ще напишем първата си прог­рама. Ще се запознаем с това какво е програмиране и каква е връзката му с компю­трите и програмните езици. Накратко ще разгледаме основните етапи при писането на софтуер. Ще въведем езика Java и ще се запознаем с Java платформата и Java технологиите. Ще разгледаме какви помощни сред­ства са ни необходими, за да можем да програмираме на Java. Ще използваме Java, за да напишем първата си програма, ще я компилираме и изпълним както от командния ред, така и от среда за разработка Eclipse. Ще се запознаем с документацията на Java, която позволява по-ната­тъшно изследване на възможностите на езика.

Глава 2. Примитивни типове и променливи

В главата "Примитивни типове и променливи" ще разгледаме примитив­ните типове и променливи в Java – какво представляват и как се работи с тях. Първо ще се спрем на типовете данни – целочислени типове, реални типове с плаваща запетая, булев тип, символен тип, обектен тип и стрингов тип. Ще продължим с това какво е променлива, какви са нейните характеристики, как се декларира, как се присвоява стойност и какво е инициализация на променлива. Ще се запознаем и с другото наимено­вание на променливата, а по-точно – "идентификатор". Към края на главата ще се спрем на литералите.

Глава 3. Оператори и изрази

В главата "Оператори, изрази" ще се запознаем с операторите и действи­ята, които те извършват върху различните типове данни. Ще разясним приоритета на операторите и групите оператори според броя на аргумен­тите, които приемат и това какво действие извършват. След това ще разгледаме преобразуването на типове, защо е нужно и как да се работим с него. Накрая ще разясним какво представляват изразите и как се използват.

Глава 4. Вход и изход от конзолата

В главата "Вход и изход от конзолата" ще се запознаем с конзолата. Ще обясним какво пред­ставлява тя, кога и как се използва, какви са принци­пите на пове­чето програмни езици за достъп до конзо­лата. Ще се запоз­наем с някой от възможностите на Java за взаимодей­ствие с потребителя. Ще разгледаме основните потоци за входно-изходни операции System.in, System.out и System.err, класът Scanner и използ­ването на формати­ращи низове за отпечатване на данни в различни формати.

Глава 5. Условни конструкции

В главата "Условни конструкции" ще разгледаме условните конструкции в Java, чрез които можем да изпълняваме различни действия в зависимост от някакво условие. Ще обясним синтаксиса на условните оператори: if и if-else с подходящи примери и ще разясним практичес­кото приложение на опера­тора за избор switch. Ще обърнем внимание на добрите прак­тики, които е нужно да бъдат следвани, с цел постигане на по-добър стил на програмиране при изпол­зването на вложени и други видове условни конструкции.

Глава 6. Цикли

В главата "Цикли" ще разгледаме конструкциите за цикли, с които можем да изпълняваме даден фрагмент програмен код многократно. Ще разгле­даме как се реализират повторения с условие (while и do-while цикли) и как се работи с for-цикли. Ще дадем примери за различните възмож­ности за дефиниране на цикъл, за начина им на конструиране и за някои от основните им приложения. Накрая ще разгледаме как можем да използ­ваме няколко цикъла един в друг (вложени цикли).

Глава 7. Масиви

В главата "Масиви" ще се запознаем с масивите като средства за обра­ботка на поредица от еднакви по тип елементи. Ще обясним какво представляват масивите, как можем да декларираме, съз­даваме и инициа­лизираме масиви. Ще обърнем внимание на едномерните и много­мерните масиви. Ще разгледаме различни начини за обхождане на масив, четене от стандартния вход и отпечатване на стандартния изход. Ще дадем много примери за задачи, които се решават с използването на масиви и ще ви покажем колко полезни са те.

Глава 8. Бройни системи

В главата "Бройни системи" ще разгледаме начините на работата с различни бройни системи и представянето на числата в тях. Повече внимание ще отделим на представянето на числата в десетична, двоична и шестнаде­сетична бройна система, тъй като те се използват масово в компютърната техника и в програмирането. Ще обясним и начините за кодиране на числовите данни в компютъра и видовете кодове, а именно: прав код, обратен код, допълнителен код и двоично-десетичен код.

Глава 9. Методи

В главата "Методи" ще се запознаем подробно с това какво е метод и защо се използват методи. Ще разберем как се декларират методи и какво е сигнатура на метод. Ще научим как да създадем собствен метод и съответно как да го използваме (извикваме) в последствие. Ще разбе­рем как можем да използваме параметри в методи и как да върнем резултат от метод. Накрая ще препоръчаме някои утвърдени практики при работата с методи.

Глава 10. Рекурсия

В главата "Рекурсия" ще се запознаем с рекурсията и нейните приложе­ния. Рекурсията представлява мощна техника, при която един метод извиква сам себе си. С нея могат да се решават сложни комбинаторни задачи, при които с лекота могат да бъдат изчерпвани различни комбина­торни конфи­гурации. Ще ви покажем много примери за правилно и неправилно използване на рекурсия и ще ви убедим колко полезна може да е тя.

Глава 11. Създаване и използване на обекти

В главата "Създаване и използване на обекти" ще се запоз­наем накратко с основните понятия в обектно-ориентираното програми­ране – класовете и обектите – и ще обясним как да използваме класовете от стандартните библиотеки на Java. Ще се спрем на някои често използ­вани системни класове и ще видим как се създават и използват техни инстанции (обекти). Ще разгледаме как можем да осъществяваме достъп до полетата на даден обект, как да извикваме конструктори и как да работим със статичните полета в класовете. Накрая ще се запознаем с понятието пакети – какво ни помагат, как да ги включваме и използваме.

Глава 12. Обработка на изключения

В главата "Обработка на изключения" ще се запознаем с изключенията в Java и обектно-ориентираното програмиране. Ще се научим как да ги прихващаме чрез конструкцията try-catch, как да ги предаваме на предходните методи чрез throws и как да хвърляме соб­ствени или прихванати изключения. Ще дадем редица примери за използ­ването им. Ще разгледаме типовете изключения и йерархията, която образуват. Накрая ще се запознаем с предимствата при използва­нето на изключения и с това как най-правилно да ги прилагаме в кон­кретни ситуации.

Глава 13. Символни низове

В главата "Символни низове" ще се запознаем със символ­ните низове: как са реализирани те в Java и по какъв начин можем да обработваме текстово съдържание. Ще прегледаме различни методи за манипулация на текст; ще научим как да извличаме поднизове по зададени параметри, как да търсим за ключови думи, както и да отделяме един низ по разделители. Ще се запознаем с методи и класове за по-елегантно и стриктно формати­ране на текстовото съдържание на конзо­лата, с различни методики за извеждане на числа, а също и с извежда­нето на отделни компоненти на текущата дата. Накрая ще предоставим полезна информация за регу­лярните изрази и ще научим по какъв начин да извличаме данни, отговарящи на определен шаблон.

Глава 14. Дефиниране на класове

В главата "Дефиниране на класове" ще разберем как можем да дефини­раме собствени класове и кои са елементите на класовете. Ще се научим да декларираме полета, конструктори и свойства в класовете. За използваната терминология

Тъй като настоящият текст е на български език, ще се опитаме да ограни­чим употребата на английски термини, доколкото е възможно. Съществу­ват обаче основателни причини да използваме и англий­ските тер­мини наред с българските им еквиваленти:

-     По-голямата част от техническата документация за Java е на английски език (повечето книги и в частност официалната докумен­тация) и затова е много важно читателите да знаят английския еквивалент на всеки използван термин.

-     Много от използваните термини не са пряко свързани с Java и са навлезли отдавна в програмисткия жаргон от английски език (напри­мер "дебъгвам", "компилирам" и "плъгин"). Тези термини ще бъдат изписвани най-често на кирилица.

-     Някои термини (например "framework" и "deployment") са трудно преводими и трябва да се използват заедно с оригинала в скобки. В настоящата книга на места такива термини са превеждани по раз­лични начини (според контекста), но винаги при първо срещане се дава и оригинални­ят термин на английски език.

Как възникна тази книга?

Често се случва някой да ме попита от коя книга да започне да се учи на програмиране. Срещат се ентусиазирани младежи, които искат да се учат да програмират, но не знаят от къде да започнат. За съжаление и аз не знам коя книга да им препоръчам. Сещам се за много книги за Java – и на български и на английски, но никоя от тях не учи на програмиране. Няма много книги (особено на български език), които да учат на концепциите на програмирането, на алгоритмично мислене, на структури от данни. Има книги за начинаещи, които учат на езика Java, но не и на основите на прог­рамирането. Има и няколко хубави книги за програмиране на бъл­гарски език, но са вече остарели. Знам няколко такива книги за C и Паскал, но не и за Java или C#. В крайна сметка няма хубава книга, която горещо да препоръчам на всеки, който иска да се захване с програ­миране от нулата.

Липсата на хубава книга по програмиране за начинаещи в един момент ни мотивира да се хванем и да напишем такава книга. Решихме, че можем да помогнем и да дадем знания и вдъхновение на много млади хора да се захванат сериозно с програмиране.

Събиране на авторския екип

Работата по книгата започва в един топъл летен ден, когато основният автор Светлин Наков, вдъхновен от идеята за написване на учебник за курсовете по "Въведение в програмирането" събира екип от двадесетина млади софту­ерни инженери, ентусиасти, които имат желание да споделят знанията си и да напишат по една глава от книгата.

Светлин Наков дефинира учебното съдържание и го разделя в глави и създава шаблон за съдържанието на всяка глава. Шаблонът съдържа структурата на текста – всички основни заглавия в дадената глава и всички подзаглавия. Остава да се напишат текста, примерите и задачите.

На първата среща на екипа учебното съдържание претърпява малко промени. По-обемните глави се разделят на няколко отделни части (например структурите от данни), възникват няколко нови глави (напри­мер работа с изключения) и се определят автори и редактори за всяка глава. Идеята е проста: всеки да напише по 1 глава от книгата и накрая да ги съединим. За да няма голяма разлика в стиловете, форматирането и начина на представяне на информацията авторите приемат единно ръководство на писателя, в което строго се описват всички правила за писане. В крайна сметка всеки си има тема и писането започва.

За проекта се създава сайт за съвместна работа в екип в Google Code на адрес http://code.google.com/p/introjavabook/, където стои последната версия на всички текстове и материали по книгата.

Задачите и сроковете

Както във всеки проект, след разпределяне на задачите се слагат крайни срокове за всяка от тях, за да се планира работата във времето. По план книгата трябва да излезе от печат през октомври, но това не се случва в срок, защото много от авторите се забавят, а някои въобще не изпълняват поетия ангажимент.

Когато идва първия краен срок едва половината от авторите са готови на време. Сроковете се удължават и голяма част от авторите завършват работата по своята глава. Започва работата на редакторите. Паралелно някои автори дописват. За някои глави се търсят нови автори, защото оригиналният автор се проваля и бива отстранен.

Няколко месеца по-късно книгата е готова на 90%, авторите загубват ентусиазъм работата започва да върви много бавно и мъчно. Светлин Наков се опитва да компенсира и да дописва недовършените теми, но работата е много. Въпреки 30-те часа, които той влага като труд всеки свободен уикенд, работата е много и все не свършва месеци наред.

Всички автори подценяват сериозно обема на работата и това е основно причината за забавянето на нейната поява. Авторите си мислят, че писането става бързо, но истината е, че за една страница текст (четене, писане, редактиране, преправяне и т.н.) отива средно по 1 час работа, та дори и повече. Сумарно за написването на цялата книга са вложени около 800-1000 работни часа труд, разпределени сред всички автори и редак­тори, което се равнява на над 6 месеца работа на един автор на пълен работен ден. Понеже всички автори пишеха в свободното си време, рабо­тата вървеше бавно и отне 4-5 месеца.
